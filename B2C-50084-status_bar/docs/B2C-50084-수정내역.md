# B2C-50084: 수정 내역

## 수정 방식
Modal 내부에 `SafeAreaProvider`를 추가하여 각 Modal Window에서 직접 safe area insets를 측정하도록 변경.
상태바 영역 배경색을 `Color.white` (또는 화면별 기존 배경색)로 설정하여 반투명 방지.

## 수정 파일 (8개)

| 화면 | 파일 | 수정 내용 |
|------|------|-----------|
| 단지 상세 > 위치 | `RoadMapViewModal.tsx` | SafeAreaProvider 추가, 배경 `Color.white` |
| 학교 상세 > 위치 | `RoadMapViewModalZuix2.tsx` | SafeAreaProvider 추가, 배경 `Color.white` |
| 단지 상세 > VR 홈투어 | `LanHomeTourModal.tsx` | SafeAreaProvider 추가, 배경 기존 `rgba(0,0,0,0.8)` 유지 |
| 단지 상세 > 부동산 계산기 | `CalculatorModal/index.tsx` | SafeAreaProvider 추가, 배경 `Color.white` |
| 분양 > 단지 정보 더보기 | `AptDetailInfoModal.tsx` | SafeAreaProvider 추가, 배경 `Color.white` |
| 집 내놓기 > 소개 페이지 | `ItemIntroModal.tsx` | SafeAreaProvider 추가 (Modal 전체 감싸는 구조), 배경 `Color.white` |
| 방 내놓기 > 주소 검색 | `PostCodeSearchModal/index.native.tsx` | SafeAreaProvider 추가, 배경 `Zuix2.Color.white` |
| 상가 > 이미지 선택 | `ImageViewerModal.tsx` | `statusBarHeight` Android 반환값 0 → `ViewDimension.statusBarHeight`로 수정 |

| `60d17fdb56` | ItemIntroModal SafeAreaProvider를 Modal 전체 감싸도록 수정 |

## 특이사항
- `ItemIntroModal`은 `Wrapper`가 `position: absolute`로 전체를 덮는 구조라서 SafeAreaProvider를 TopBar만 감싸면 터치/스크롤이 차단됨. Modal 전체 콘텐츠를 감싸는 구조로 변경.
- `ImageViewerModal`은 SafeAreaView를 사용하지 않고 수동 `paddingTop`을 사용. Android에서 `return 0`이던 값을 `ViewDimension.statusBarHeight`로 수정.
