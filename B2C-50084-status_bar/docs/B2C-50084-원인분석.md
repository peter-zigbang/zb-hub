# B2C-50084: 상태바 겹침 원인 분석

## 현상
- Galaxy S26 Ultra (Android 16)에서 위치보기 등 Modal 화면의 상단 네비게이션 바가 시스템 상태바와 겹쳐 표시
- S24 Ultra (Android 16), S20+ (Android 12) 등 다른 기기에서는 정상

## 근본 원인
React Native의 `Modal`은 새로운 Android Window를 생성한다. 앱 루트(`app.tsx`)의 `SafeAreaProvider`는 메인 Window 기준으로 insets를 측정하는데, Modal 내부의 `SafeAreaView`/`useSafeAreaInsets()`가 이 값을 그대로 사용한다.

대부분의 기기에서는 메인 Window와 Modal Window의 insets가 동일하여 문제가 없지만, S26 Ultra는 디스플레이 하드웨어(카메라 홀 크기/위치, 디스플레이 컷아웃 구성)가 달라 insets 불일치가 발생한다.

## 핵심
- 코드 자체는 원래 잘못된 것 (Modal 내부에 `SafeAreaProvider` 누락)
- S26 Ultra가 이 잠재 버그를 처음 드러낸 기기
- 동일 프로젝트 내 `TotalMessageSearchScreen.tsx`에서는 올바른 패턴 사용 중이었음

## 참고
- `targetSdkVersion = 35`, React Native 0.78.2
- AGP 8.8.0 (React Native Gradle Plugin 제공)
